nginx-deployment
kind: Deployment
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:latest
          ports:
            - containerPort: 80
🟢
nginx-pod
kind: Pod
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
  labels:
    app: web
spec:
  containers:
    - name: nginx-container
      image: nginx:latest
      ports:
        - containerPort: 80
🔀
nginx-service
kind: Service
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
      nodePort: 30007
  type: NodePort
📦 Deployment
  • 3 réplicas de nginx corriendo simultáneamente
  • Auto-recuperación si un pod falla
  • Selector: app: nginx
  • Puerto del contenedor: 80
🔀 Service (NodePort)
  • Expone la app en el puerto externo 30007
  • Balancea carga entre los 3 pods automáticamente
  • Protocolo TCP, puerto interno 80
  • Acceso: http://<node-ip>:30007
🟢 Pod independiente
  • Pod aislado nginx-pod para pruebas
  • Label: app: web
  • Imagen nginx:latest en puerto 80
  • Útil para debugging sin afectar el Deployment
⚙️ Comandos para aplicar
  • kubectl apply -f deployment.yaml
  • kubectl apply -f pod.yaml
  • kubectl apply -f service.yaml
  • kubectl get pods para verificar
